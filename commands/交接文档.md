# 生成交接文档

用户调用了 `/交接文档`。这个手动入口在 Hook 未信任、被禁用或失败时也必须正常工作。

请执行与 `/handoff` 完全相同的流程：

1. 读取本插件的 `commands/handoff.md`。
2. 按其中的要求检查当前工作区。
3. 按安全排除规则检查工作区，在当前工作区根目录原子生成 `HANDOFF.md`；不得读取或复制凭据、`.env` 值、原始 transcript、日志、数据库、隐藏推理或截图。
4. 运行插件脚本的 `scan` 命令；发现高置信秘密时停止，不创建新任务。
5. 扫描通过后计算 SHA-256，并按 `list_projects → list_threads → create_thread → set_thread_title → read_thread` 的顺序创建和验证干净的新任务；禁止使用 `fork_thread`。
6. 新任务提示第一句必须是 `Read HANDOFF.md first and continue the project.`，并包含绝对路径、SHA-256 和大小允许时的完整内容。
7. 新任务标题继承原任务并加 `（续接 N）`。项目未登记时使用 projectless 目标。
8. 如果任务工具不可用或任一步骤未验证，请只报告生成文件的绝对路径，不得假装已完成 UI 点击、附件导入、创建、改名或导航。

最终必须保存文件名：`HANDOFF.md`。
